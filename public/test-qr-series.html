Тестирование QR-кодов для серии шаблонов

Сценарий 1: Оригинальный создатель серии

Ситуация: Пользователь создал серию из 10 шаблонов

Ожидаемое поведение: Показывается QR-код для клиентов (получение шаблона)

После сканирования клиентом: series_current уменьшается на 1

Сценарий 2: Клиент получил шаблон из серии

Ситуация: Клиент отсканировал QR и получил шаблон "(Получено по QR)"

Ожидаемое поведение: Показывается QR-код для деактивации

После сканирования создателем: Шаблон помечается как "Использован"

Сценарий 3: Просмотр серии другими пользователями

Ситуация: Другие пользователи смотрят на серию

Ожидаемое поведение: Видят актуальный счетчик использований

Пример: "Доступно: 9 из 10" (если 1 уже получен)

Проверочный список изменений:

  • ✅ Добавлена логика показа QR-кода для полученных шаблонов
  • ✅ Обновлена деактивация для разных типов шаблонов
  • ✅ Добавлены публичные маршруты для QR-кодов

Следующие шаги:

  2. Протестировать создание серии и получение шаблона по QR
  3. Проверить корректность обновления счетчика серии
  4. Убедиться в работе деактивации шаблонов

Пример URL для тестирования

Получение шаблона: http://tyty/qr/claim/TOKEN

Деактивация: http://tyty/qr/deactivate/TOKEN

Статус QR: GET /template-qr/ID/status

Генерация клиент QR: POST /template-qr/ID/generate-client

Генерация создатель QR: POST /template-qr/ID/generate-creator
